A Bidirectional LSTM (BiLSTM) reads a sequence twice — left→right and right→left — then fuses the two views. This simple idea consistently improves token-level tasks (tagging, per-time-step regression/classification) when both past and future context are relevant. Below, you’ll get: clear intuition, the core math, when BiLSTMs shine vs. Transformers, and a step‑by‑step walkthrough of your provided CNN‑BiLSTM code that performs per‑base predictions on RNA sequences. We close with hyperparameter tips, debugging checklists, and a time/space complexity analysis.

Why BiLSTM?
A standard LSTM is causal: at position t, it only sees x_t. That’s perfect for streaming, but many problems need future evidence, e.g.:

NLP tagging (POS, NER): the word after often clarifies the tag of the current word.
Bioinformatics: base‑pairing or loop context downstream can refine the signal at the current base.
Time series diagnostics: a local spike may only be recognized with a small look‑ahead.
When to prefer BiLSTM (vs. Transformers or causal LSTM)
Use BiLSTM when:

You need per‑token predictions, and future context helps.
Data is small to medium; you want a compact, stable baseline.
Latency permits a full pass over the sequence (i.e., not streaming).
Prefer Transformers when:

You need long‑range, global interactions over very long sequences, and you have ample data/compute.
Prefer forward (causal) LSTM when:

You must predict online/streaming with strict causality.
Architecture pattern used here:
CNN + BiLSTM
Your code couples a Conv1d over features with a BiLSTM:

Local mixing with Conv1d: small kernels (e.g., size 5) blend neighboring features, creating richer local descriptors.
Global context with BiLSTM: processes the full sequence forward and backward to capture long‑range dependencies.
Per‑position head: a linear layer maps each time step’s contextualized vector to the target(s).
This hybrid is a strong, lightweight alternative to deep stacks of recurrent layers.

def collate_train(batch):
    maxL = max(s.L for s in batch)
    X = np.zeros((B, maxL, F), dtype=np.float32)
    Y = np.zeros((B, maxL, 5), dtype=np.float32)
    mask = np.zeros((B, maxL), dtype=np.float32)

    for i, s in enumerate(batch):
        X[i, :s.L] = s.X
        Y[i, :s.scored_len] = s.Y
        mask[i, :s.scored_len] = 1.0

    return {'X': torch.from_numpy(X),
            'Y': torch.from_numpy(Y),
            'mask': torch.from_numpy(mask)}

Standardization of targets
The script computes a dataset‑wide mean/std per target and standardizes Y before training, then inverses at inference:

@dataclass
class Scaler:
    mean: np.ndarray
    std: np.ndarray
    def transform(self, y): return (y - self.mean) / np.clip(self.std, 1e-6, None)
    def inverse(self, y):   return y * np.clip(self.std, 1e-6, None) + self.mean
Why it’s important:

Improves optimization (Adam + plateau scheduler converge more reliably).
Prevents scale dominance across targets.
The model: CNNBiLSTM
class CNNBiLSTM(nn.Module):
    def __init__(self, in_dim=70, cnn_filters=64, cnn_kernel=5, lstm_hidden=64, dropout=0.2, num_targets=5):
        super().__init__()
        self.conv = nn.Conv1d(in_channels=in_dim, out_channels=cnn_filters,
                              kernel_size=cnn_kernel, padding=cnn_kernel//2)
        self.relu = nn.ReLU(inplace=True)
        self.drop = nn.Dropout(dropout)
        self.lstm = nn.LSTM(input_size=cnn_filters, hidden_size=lstm_hidden,
                            batch_first=True, bidirectional=True)
        self.head = nn.Linear(2*lstm_hidden, num_targets)

    def forward(self, x):          # x: (B, L, F)
        x = x.transpose(1, 2)      # (B, F, L)
        x = self.drop(self.relu(self.conv(x)))  # (B, C, L)
        x = x.transpose(1, 2)      # (B, L, C)
        x, _ = self.lstm(x)        # (B, L, 2H)
        x = self.drop(x)
        return self.head(x)        # (B, L, num_targets)
Design notes:

Conv1d with padding preserves length, mixing local neighborhoods.
Bidirectional LSTM contextualizes each position using both past/future.
Dropout regularizes both conv outputs and recurrent outputs (note PyTorch’s nn.LSTM(dropout=…) only applies when num_layers > 1; here dropout is applied externally).
Parameter count (default hyperparams).

Conv1d: 64 x(70 x 5) + 64 = 22,464
LSTM (per direction): 4H x (C + H) + 2 x 4H biases = 4 * 64 * (64+64) + 512 = 33,280 → both dirs: 66,560
Linear head: 128 x 5 + 5 = 645 → Total ≈ 89,669 parameters — compact and fast.
Loss: masked MSE for token‑level regression
Only scored positions contribute to the loss:

def masked_mse(pred, target, mask):
    mask = mask.unsqueeze(-1)           # (B,L,1)
    diff = (pred - target) ** 2 * mask  # (B,L,5)
    denom = torch.clamp(mask.sum() * pred.size(-1), min=1.0)
    return diff.sum() / denom

class BiLSTMRegressor(nn.Module):
    def __init__(self, in_dim, hidden=128, out_dim=1, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(in_dim, hidden, num_layers=1,
                            bidirectional=True, batch_first=True)
        self.dropout = nn.Dropout(dropout)
        self.head = nn.Linear(2*hidden, out_dim)

    def forward(self, x, lengths=None):
        # x: (B, L, F)
        if lengths is not None:
            packed = nn.utils.rnn.pack_padded_sequence(
                x, lengths, batch_first=True, enforce_sorted=False)
            y, _ = self.lstm(packed)
            y, _ = nn.utils.rnn.pad_packed_sequence(y, batch_first=True)
        else:
            y, _ = self.lstm(x)
        return self.head(self.dropout(y))  # (B, L, out_dim)
